2. 文件类型（根据文档进行开发） #

详细配置见小程序管理工具的框架模块 #

  • .json配置文件 #

  • .js编写业务逻辑文件 #

  • .wxml编写页面文件 #

  • .wxss编写页面样式 #

  • pages文件夹下的是配置单页的，根目录下的是配置全局的。 #

页面级别的配置会覆盖全局的配置 #

3. 生命周期函数（某一时刻自动执行的函数） #

全局： #

  • onLaunch //小程序启动时执行（中途退出，时间很短，不会重新启动） #

    参数options： #

    scene： 从什么场景进入小程序 #

    path: 启动页面路径 #

    query： 启动小程序携带的参数，启动参数 #

  • onShow //在小程序重新展示时执行 #

    参数同上 #

  • onHide //在小程序取消展示时执行 #

  • onError //在脚本执行错误时执行 #

4. app.js中写全局的方法和数据 #

全局数据在 globalData 对象中写 #

在页面级别 js 文件中可通过 #

const app = getApp( ) 获取应用实例 #

app.方法名（ 变量名 ） #

使用全局中的方法和数据 #

5. 生命周期函数 #

页面级别： #

onLoad //页面第一次加载时 #

onShow //页面展示时 #

onReady //页面第一次加载并渲染完成时 #

onHide //页面取消展示时 #

onPullDownRefresh //下拉刷新时 #

onReachBottom //滚动到最底部时 #

（ 更多生命周期见小程序管理工具文档——框架——框架接口 ） #

6. 页面级别展示数据 #

数据必须定义在 js 文件的data对象中，在wxml中用 解析变量 #

7. 绑定方法 #

bindtap="方法名" #

在js文件中定义方法 #

8. 改变data对象中定义的变量 #

用内置的方法: setData( ) 在方法中使用时注意加this #

this.setData( { #

变量名：值 #

} ) #

9. 条件指令 #

wxml组件上添加 wx : if = " { { 变量名 } } " #

会销毁元素 #

10. 循环指令 #

wx : key = " *this " 循环必须加key，值要不同 #

wx : for = " { { 数组 } } " #

item 表示每一项，默认 #

index 下标，从零开始 #

wx : for - item = " 自定义每一项名字 " #

wx : for - index= " 自定义下标名字 " #

可用 包裹多个需要循环的结构 #

11. 小程序中用rpx，默认全宽为750rpx #

要看清楚开发时的设计稿是以多少为基准的，如果是 375 px，则相应尺寸要乘以 2 #

一定要检查单位是不是 rpx #

12. 小程序自带一些图标组件，表单事件绑定的回调函数有事件对象形参，可获取相关数据 #
